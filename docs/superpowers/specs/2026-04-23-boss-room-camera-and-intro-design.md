# ボス部屋カメラ・導入演出・雑魚AI 調整設計

## 目的

ボス部屋で次の3点を改善する。

1. ボス部屋にいる雑魚敵がプレイヤーと離れたときに停止しないこと
2. ボス戦カメラがプレイヤーとボスの中間を映しすぎて、両者とも見えなくなる状況を減らすこと
3. ボス登場演出を、見上げから始まるより印象的な導入に差し替えること

## 対象ファイル

- `project/game/enemy/Enemy.cpp`
- `project/game/enemy/Enemy.h`
- `project/game/scene/GamePlayScene.cpp`
- `project/game/enemy/BossManager.h`
- 必要なら `project/game/enemy/BossManager.cpp`

既に変更済みの `Boss.cpp` の降下演出は活かしつつ、必要以上の再変更は避ける。

## 設計方針

### 1. ボス部屋の雑魚AI

通常フロアと同じ追跡距離だと、ボス部屋の広さに対して `chaseRange_` が短く、雑魚が `Patrol` や短い `Investigate` に戻りやすい。

そのため、ボス部屋では以下のどちらも有効にする。

- プレイヤーを認識する距離を通常より広くする
- 一度認識したあとの `investigateTimer_` を通常より長く維持する

実装上は、`Enemy::DecideNextState()` と `Enemy::UpdateChasePlayer()` の参照距離に、ボス部屋専用の有効追跡距離を使う。
通常フロアの挙動を壊さないため、分岐はボス部屋時のみ有効にする。

### 2. ボス戦カメラ

現状はプレイヤーとボスの中点を基準にしているため、距離が大きいほどカメラが引きすぎる。

改善後は「プレイヤー主役」の構図に変える。

- カメラの注視中心はプレイヤー寄りに置く
- ボス方向へは一定量だけオフセットする
- プレイヤーとボスの距離が広がっても、後退量と高さに上限を設ける

これにより、ボスが遠いときでもプレイヤーを見失いにくくする。
ボスは常に完全に画面内へ収めることを目的にせず、主役であるプレイヤーの視認性を優先する。

### 3. ボス登場演出

導入は既存の3段階を拡張し、4段階にする。

1. `SkyLook`
プレイヤー付近から上空を見上げる。何かが来る予感を作る。

2. `BossReveal`
上空のボスを捉える。ボスが空中にいる姿をはっきり見せる。

3. `BossDropFollow`
ボスの落下に合わせてカメラが追従する。主役はボス。

4. `BossLandImpact`
着地直後に短いワンアクションを入れる。
内容は「少し寄る」「一瞬低く煽る」「ごく短いシェイク」の組み合わせで、長さは短く保つ。

5. `ToBattle`
戦闘カメラへ滑らかに復帰する。

既存の `IntroCameraState` を増やすか、意味を差し替える形で対応する。
実装は `GamePlayScene.cpp` のカメラ更新部に寄せる。

## 期待する挙動

- ボス部屋の雑魚は、プレイヤーが離れてもすぐ停止せず追い続ける
- ボス戦中、プレイヤーが画面の主役として見やすい
- ボス登場時は、まず空を見上げてから上空のボスを見せ、落下を追い、着地で一発見せ場を作ってから戦闘へ戻る

## 非目的

- 雑魚AI全体の大規模なリファクタ
- 通常フロアのカメラ仕様変更
- ボスの攻撃AIやダメージ計算の調整

## テスト観点

- ボス部屋でプレイヤーが大きく移動しても、雑魚が停止せず追跡または探索継続するか
- ボスとプレイヤーが離れたときでも、プレイヤーが見切れにくいか
- 登場演出が `見上げ -> 上空のボス -> 落下追従 -> 着地演出 -> 戦闘カメラ` の順で自然につながるか
- 既存のボス導入中停止仕様が壊れていないか
